## 1. 기본편: Neovim 시작하기

### 1-1. 시작과 종료

```bash
# 시작
nvim                    # 빈 화면으로 시작
nvim 파일명             # 특정 파일 열기
nvim 파일1 파일2        # 여러 파일 열기 (버퍼로 관리)
nvim +100 파일명        # 파일의 100번째 줄에서 시작

# 종료 (Neovim 안에서)
:q                      # 종료 (변경 없을 때)
:q!                     # 강제 종료 (변경사항 버림)
:w                      # 저장
:wq 또는 :x 또는 ZZ    # 저장 후 종료
:qa                     # 모든 버퍼 종료
```

### 1-2. 모드 이해하기

Neovim은 **모드**에 따라 키의 동작이 달라집니다.

```
                    i, a, o
        ┌─────── Insert 모드 ◄─────────┐
        │        (글자 입력)            │
        │                              │
        ▼   ESC                        │
   Normal 모드 ────────────────► Visual 모드
   (명령 실행)      v, V, Ctrl+v      (텍스트 선택)
        │                              │
        │         :                    │
        └──────► Command 모드          │
                (: 명령 입력)          │
                       ESC             │
                        └──────────────┘
```

| 모드 | 진입 방법 | 용도 | 하단 표시 |
|------|----------|------|----------|
| **Normal** | `ESC` | 이동, 복사, 삭제, 명령 실행 | -- NORMAL -- |
| **Insert** | `i`, `a`, `o` | 텍스트 입력 | -- INSERT -- |
| **Visual** | `v`, `V`, `Ctrl+v` | 텍스트 선택 | -- VISUAL -- |
| **Command** | `:` | Vim 명령어 입력 | 하단에 `:` 표시 |

#### Normal 모드에서 Insert 모드로 진입하는 방법들
| 키 | 설명 |
|----|------|
| `i` | 커서 **앞**에서 입력 시작 (가장 많이 사용) |
| `a` | 커서 **뒤**에서 입력 시작 |
| `I` | 줄 **맨 앞**에서 입력 시작 |
| `A` | 줄 **맨 뒤**에서 입력 시작 |
| `o` | 현재 줄 **아래**에 새 줄 만들고 입력 |
| `O` | 현재 줄 **위**에 새 줄 만들고 입력 |

> **핵심:** 뭔가 잘못되면 `ESC`를 누르세요. Normal 모드로 돌아갑니다.

### 1-3. 기본 이동

Normal 모드에서의 커서 이동:

#### 한 글자/줄 이동
| 키 | 설명 |
|----|------|
| `h` | 왼쪽 (← 화살표와 동일) |
| `j` | 아래 (↓ 화살표와 동일) |
| `k` | 위 (↑ 화살표와 동일) |
| `l` | 오른쪽 (→ 화살표와 동일) |

> 화살표 키도 동작하지만, `hjkl`에 익숙해지면 손이 홈로우에서 떠나지 않아 훨씬 빠릅니다.

#### 단어 단위 이동
| 키 | 설명 |
|----|------|
| `w` | 다음 단어 시작으로 이동 |
| `b` | 이전 단어 시작으로 이동 |
| `e` | 현재/다음 단어 끝으로 이동 |

#### 줄 내 이동
| 키 | 설명 |
|----|------|
| `0` | 줄 맨 앞으로 |
| `^` | 줄의 첫 글자로 (공백 제외) |
| `$` | 줄 맨 뒤로 |

#### 화면/파일 이동
| 키 | 설명 |
|----|------|
| `gg` | 파일 맨 위로 |
| `G` | 파일 맨 아래로 |
| `100G` 또는 `:100` | 100번째 줄로 이동 |
| `Ctrl+d` | 반 페이지 아래로 스크롤 |
| `Ctrl+u` | 반 페이지 위로 스크롤 |
| `Ctrl+f` | 한 페이지 아래로 |
| `Ctrl+b` | 한 페이지 위로 |
| `%` | 매칭되는 괄호로 이동 |

### 1-4. 기본 편집

Normal 모드에서:

#### 삭제
| 키 | 설명 |
|----|------|
| `x` | 커서 위의 한 글자 삭제 |
| `dd` | 현재 줄 전체 삭제 (잘라내기) |
| `dw` | 커서부터 단어 끝까지 삭제 |
| `d$` 또는 `D` | 커서부터 줄 끝까지 삭제 |
| `5dd` | 현재 줄부터 5줄 삭제 |

#### 복사/붙여넣기
| 키 | 설명 |
|----|------|
| `yy` | 현재 줄 복사 (yank) |
| `yw` | 단어 복사 |
| `5yy` | 5줄 복사 |
| `p` | 커서 뒤에 붙여넣기 |
| `P` | 커서 앞에 붙여넣기 |

#### 실행 취소/다시 실행
| 키 | 설명 |
|----|------|
| `u` | 실행 취소 (Undo) - `Ctrl+Z`와 같은 역할 |
| `Ctrl+r` | 다시 실행 (Redo) |

#### 검색
| 키 | 설명 |
|----|------|
| `/검색어` | 아래 방향으로 검색 |
| `?검색어` | 위 방향으로 검색 |
| `n` | 다음 검색 결과로 이동 |
| `N` | 이전 검색 결과로 이동 |
| `*` | 커서 아래 단어를 검색 |
| `ESC ESC` | 검색 하이라이트 끄기 |

#### 치환 (찾아 바꾸기)
```vim
:s/old/new/         " 현재 줄에서 첫 번째 'old'를 'new'로
:s/old/new/g        " 현재 줄에서 모든 'old'를 'new'로
:%s/old/new/g       " 파일 전체에서 모든 'old'를 'new'로
:%s/old/new/gc      " 파일 전체에서 하나씩 확인하며 바꾸기
```

### 1-5. 창(Window) 분할

```vim
:split 또는 :sp       " 수평 분할 (위/아래)
:vsplit 또는 :vsp      " 수직 분할 (좌/우)
:sp 파일명            " 파일을 새 수평 분할로 열기
:vsp 파일명           " 파일을 새 수직 분할로 열기
```

**창 이동 (Normal 모드):**
| 키 | 설명 |
|----|------|
| `Ctrl+h` | 왼쪽 창으로 |
| `Ctrl+j` | 아래 창으로 |
| `Ctrl+k` | 위 창으로 |
| `Ctrl+l` | 오른쪽 창으로 |
| `Ctrl+w q` | 현재 창 닫기 |

### 1-6. 버퍼 (여러 파일 다루기)

Neovim에서 여는 각 파일은 **버퍼**로 관리됩니다.
상단 탭바(lualine)에 열린 버퍼 목록이 표시됩니다.

| 키/명령 | 설명 |
|---------|------|
| `,q` | 이전 버퍼로 이동 |
| `,w` | 다음 버퍼로 이동 |
| `,x` | 현재 버퍼 닫기 |
| `:e 파일명` | 새 파일을 버퍼로 열기 |
| `:ls` | 열린 버퍼 목록 보기 |

## 3. 플러그인별 상세 사용법

### 3-1. telescope.nvim (퍼지 검색)

> **한 줄 설명:** 파일, 텍스트, 버퍼 등을 빠르게 퍼지 검색하는 도구

#### 어떤 상황에서 쓰나?
- 수만 개의 파일 중에서 원하는 파일을 이름으로 찾고 싶을 때
- 코드 전체에서 특정 문자열을 검색하고 싶을 때 (grep)
- 열어둔 여러 버퍼 중 하나로 빠르게 전환하고 싶을 때

#### 실제 사용 시나리오

**시나리오: "AttributeEngine.cpp 파일을 열고 싶다"**
1. `,ff` 를 누릅니다
2. 검색창이 뜹니다: `> ` 커서가 깜빡임
3. `atteng`만 입력해도 `AttributeEngine.cpp`가 목록에 나타남 (퍼지 매칭)
4. `Enter`로 해당 파일을 엽니다

**시나리오: "코드에서 'getColumnValue'를 사용하는 모든 곳을 찾고 싶다"**
1. `,fg` 를 누릅니다
2. 검색창에 `getColumnValue` 입력
3. 모든 파일에서 해당 텍스트가 포함된 줄이 목록으로 표시됨
4. 오른쪽에 파일 미리보기가 표시됨
5. 원하는 결과를 선택하고 `Enter`

#### 검색창 내 조작법

검색 결과 목록이 뜬 상태에서:

| 키 | 모드 | 설명 |
|----|------|------|
| `Ctrl+j` | Insert | 다음 항목으로 이동 (아래) |
| `Ctrl+k` | Insert | 이전 항목으로 이동 (위) |
| `Enter` | Insert | 선택한 항목 열기 |
| `Ctrl+x` | Insert | 수평 분할로 열기 |
| `Ctrl+v` | Insert | 수직 분할로 열기 |
| `Ctrl+t` | Insert | 새 탭에서 열기 |
| `Ctrl+u` | Insert | 미리보기 위로 스크롤 |
| `Ctrl+d` | Insert | 미리보기 아래로 스크롤 |
| `ESC` | Insert | 검색 닫기 |
| `j` / `k` | Normal | 항목 이동 (ESC 누른 후) |

### 3-2. nvim-tree.lua (파일 탐색기)

> **한 줄 설명:** 화면 왼쪽에 디렉토리 트리를 표시하는 파일 탐색기

#### 어떤 상황에서 쓰나?
- 프로젝트의 디렉토리 구조를 탐색하고 싶을 때
- 파일을 생성, 삭제, 이름 변경하고 싶을 때
- 현재 파일 근처의 다른 파일을 열고 싶을 때

#### 실제 사용 시나리오

**시나리오: "현재 디렉토리에 있는 다른 파일들을 보고 싶다"**
1. `,e` 를 누릅니다 → 왼쪽에 파일 트리가 열림
2. `j`/`k`로 파일 목록을 위아래로 이동
3. `Enter`로 디렉토리 열기/닫기 또는 파일 열기
4. 다시 `,e` 를 누르면 트리가 닫힘

#### 트리 내 조작법

| 키 | 설명 |
|----|------|
| `Enter` 또는 `o` | 파일 열기 / 디렉토리 접기·펼치기 |
| `Ctrl+v` | 수직 분할로 파일 열기 |
| `Ctrl+x` | 수평 분할로 파일 열기 |
| `Tab` | 파일 미리보기 (커서는 트리에 유지) |
| `a` | 새 파일 또는 디렉토리 생성 (`/`로 끝나면 디렉토리) |
| `d` | 삭제 (확인 물어봄) |
| `r` | 이름 변경 |
| `x` | 잘라내기 (이동용) |
| `c` | 복사 |
| `p` | 붙여넣기 (잘라내기/복사 후) |
| `R` | 트리 새로고침 |
| `H` | 숨김 파일 표시/숨기기 토글 |
| `q` | 트리 닫기 |

### 3-3. nvim-cmp (자동완성)

> **한 줄 설명:** 코드 작성 중 자동으로 완성 후보를 제안하는 도구

#### 어떤 상황에서 쓰나?
- C++ 코드에서 함수나 변수 이름을 자동완성하고 싶을 때
- 멤버 변수/함수를 `.` 또는 `->` 입력 후 자동으로 보고 싶을 때
- 파일 경로를 자동완성하고 싶을 때

#### 실제 사용 시나리오

**시나리오: "객체의 멤버 함수를 자동완성하고 싶다"**
1. Insert 모드에서 `obj->` 를 입력합니다
2. 자동으로 팝업이 뜨면서 사용 가능한 멤버 목록이 표시됨
3. `Tab` 또는 `Ctrl+j`로 원하는 항목으로 이동
4. `Enter`로 확정 → 코드에 삽입됨

#### 팝업 조작법

| 키 | 설명 |
|----|------|
| `Ctrl+Space` | 팝업이 안 뜰 때 수동으로 열기 |
| `Tab` 또는 `Ctrl+j` | 다음 항목 선택 |
| `Shift+Tab` 또는 `Ctrl+k` | 이전 항목 선택 |
| `Enter` | 선택 확정 |
| `Ctrl+e` | 팝업 닫기 (무시) |
| `Ctrl+b` | 문서 설명 위로 스크롤 |
| `Ctrl+f` | 문서 설명 아래로 스크롤 |

#### 완성 소스 아이콘

팝업 항목 옆의 표시로 출처를 알 수 있습니다:
- **LSP** 표시: clangd가 분석한 함수, 변수, 타입 등
- **Buffer** 표시: 현재 파일에 있는 단어
- **Path** 표시: 파일/디렉토리 경로
- **Snippet** 표시: 코드 조각 템플릿

### 3-4. LSP - clangd (코드 인텔리전스)

> **한 줄 설명:** C++ 코드에서 IDE 수준의 코드 탐색, 분석, 수정 기능을 제공

#### 어떤 상황에서 쓰나?
- 함수의 정의를 보고 싶을 때
- 이 함수를 어디서 호출하는지 찾고 싶을 때
- 변수나 함수의 타입 정보를 보고 싶을 때
- 변수/함수 이름을 일괄 변경하고 싶을 때
- 컴파일 에러를 코딩 중에 미리 확인하고 싶을 때

#### 실제 사용 시나리오

**시나리오: "이 함수가 어디에 정의되어 있는지 보고 싶다"**
1. Normal 모드에서 함수 이름 위에 커서를 놓습니다
2. `gd` 를 누릅니다
3. 해당 함수의 정의가 있는 파일의 해당 줄로 점프합니다
4. `Ctrl+o`로 이전 위치로 돌아갑니다

**시나리오: "이 함수를 호출하는 모든 곳을 찾고 싶다"**
1. 함수 이름 위에 커서를 놓고 `gr` 을 누릅니다
2. 모든 참조 위치가 목록으로 표시됩니다
3. 원하는 위치를 선택하여 이동할 수 있습니다

**시나리오: "변수 이름을 한꺼번에 바꾸고 싶다"**
1. 변수 이름 위에 커서를 놓고 `,rn` 을 누릅니다
2. 새 이름을 입력하고 `Enter`
3. 해당 변수가 사용된 모든 곳이 자동으로 변경됩니다

#### LSP 키맵 상세

| 키 | 설명 | 돌아오기 |
|----|------|---------|
| `gd` | **정의로 이동** - 함수/변수가 선언된 곳 | `Ctrl+o` |
| `gD` | **선언으로 이동** - 헤더의 선언 | `Ctrl+o` |
| `gr` | **참조 찾기** - 이것을 사용하는 모든 곳 | 목록에서 선택 |
| `gi` | **구현으로 이동** - 인터페이스의 실제 구현 | `Ctrl+o` |
| `gt` | **타입 정의** - 변수 타입의 정의 위치 | `Ctrl+o` |
| `K` | **호버 정보** - 커서 아래 항목의 타입과 문서 표시 | 자동 닫힘 |
| `Ctrl+k` | **시그니처** - 함수 매개변수 정보 | 자동 닫힘 |
| `,rn` | **이름 변경** - 모든 참조도 함께 변경 | - |
| `,ca` | **코드 액션** - 사용 가능한 자동 수정 목록 | - |

#### 진단(에러/경고) 이동

코드에 문제가 있으면 왼쪽에 아이콘이 표시되고 해당 코드에 밑줄이 그어집니다:
-  빨간 동그라미 = 에러
-  노란 삼각형 = 경고
-  파란 `i` = 정보

| 키 | 설명 |
|----|------|
| `]d` | 다음 진단으로 점프 |
| `[d` | 이전 진단으로 점프 |
| `,dd` | 현재 줄의 진단을 팝업으로 상세히 보기 |
| `,dl` | 모든 진단 목록 열기 |

#### LSP 상태 확인

```vim
:LspInfo        " 현재 버퍼에 연결된 LSP 서버 정보
:LspLog         " LSP 로그 보기 (문제 해결용)
:LspRestart     " LSP 서버 재시작
```

### 3-5. gitsigns.nvim (git 변경 표시)

> **한 줄 설명:** 편집기 왼쪽에 git 변경 상태를 실시간으로 표시

#### 어떤 상황에서 쓰나?
- 코딩 중 어떤 줄을 수정/추가/삭제했는지 확인하고 싶을 때
- 수정한 부분으로 빠르게 이동하고 싶을 때
- 특정 변경만 스테이지하거나 되돌리고 싶을 때
- 이 줄을 마지막으로 수정한 사람을 보고 싶을 때 (git blame)

#### 화면에 뭐가 보이나?

에디터 왼쪽 기호 열(sign column)에 색깔 막대가 표시됩니다:
- **│ (초록색)**: 새로 추가된 줄
- **│ (파란색)**: 수정된 줄
- **_ (빨간색)**: 삭제된 줄 (아래쪽 줄에 표시)

#### 실제 사용 시나리오

**시나리오: "내가 수정한 부분을 순서대로 확인하고 싶다"**
1. `]c` 를 누릅니다 → 다음 변경 지점(hunk)으로 커서 이동
2. `,hp` 를 누릅니다 → 변경 전후를 팝업으로 미리보기
3. 계속 `]c`로 다음 변경으로 이동
4. `[c`로 이전 변경으로 돌아가기

**시나리오: "이 변경만 되돌리고 싶다"**
1. 되돌리고 싶은 변경 위에 커서를 놓습니다
2. `,hr` 을 누릅니다 → 해당 hunk가 git의 원래 상태로 복원됨

#### 키맵

| 키 | 설명 |
|----|------|
| `]c` | 다음 변경(hunk)으로 이동 |
| `[c` | 이전 변경(hunk)으로 이동 |
| `,hp` | 변경 미리보기 (변경 전후 비교) |
| `,hs` | 이 변경을 스테이지 (git add) |
| `,hr` | 이 변경을 리셋 (원래 상태로 복원) |
| `,hb` | 현재 줄의 git blame (누가, 언제, 무슨 커밋으로 수정했는지) |

### 3-6. vim-fugitive (git 명령어)

> **한 줄 설명:** Neovim 안에서 git 명령어를 실행하는 도구

#### 어떤 상황에서 쓰나?
- 터미널로 나가지 않고 git 작업을 하고 싶을 때
- git status, diff, blame, log를 보고 싶을 때

#### 주요 명령어

| 명령어 | 설명 |
|--------|------|
| `:Git` 또는 `:G` | git status 화면 열기 (가장 많이 사용) |
| `:Git diff` | git diff 보기 |

| `:Git log` | 커밋 로그 보기 |
| `:Git add %` | 현재 파일을 스테이지 |
| `:Git commit` | 커밋 작성 |
| `:Gdiffsplit` | 현재 파일의 diff를 좌우 분할로 보기 |

#### :Git (status) 화면에서의 조작

`:Git`을 실행하면 git status 화면이 열립니다:

| 키 | 설명 |
|----|------|
| `s` | 파일을 스테이지 (add) |
| `u` | 파일을 언스테이지 |
| `=` | diff 토글 (인라인) |
| `Enter` | 해당 파일 열기 |
| `cc` | 커밋 작성 |
| `q` | status 화면 닫기 |

### 3-7. Comment.nvim (주석 토글)

> **한 줄 설명:** 한 키로 코드 주석을 켜고 끄는 도구

#### 어떤 상황에서 쓰나?
- 코드를 일시적으로 주석처리하고 싶을 때
- 여러 줄을 한꺼번에 주석처리하고 싶을 때
- 주석을 해제하고 싶을 때

#### 사용법

| 키 | 모드 | 설명 |
|----|------|------|
| `gcc` | Normal | **현재 줄** 주석 토글 (한 번 누르면 주석, 다시 누르면 해제) |
| `gc` + 범위 | Normal | **범위**에 주석 토글 (`gcip` = 현재 문단, `gc3j` = 아래 3줄) |
| `gc` | Visual | **선택한 영역** 주석 토글 |
| `gbc` | Normal | 현재 줄 **블록 주석** 토글 (`/* ... */`) |
| `gb` | Visual | 선택 영역 **블록 주석** 토글 |

#### 예시 (C++)

**주석 전:**
```cpp
int x = 10;
int y = 20;
```

**`gcc` 한 번 (현재 줄 주석):**
```cpp
// int x = 10;
int y = 20;
```

**두 줄 선택 후 `gc` (Visual 모드 주석):**
```cpp
// int x = 10;
// int y = 20;
```

### 3-8. nvim-treesitter (향상된 구문 강조)

> **한 줄 설명:** 코드의 구문 트리를 분석하여 정확한 구문 강조를 제공

#### 어떤 상황에서 쓰나?
- **자동으로 동작합니다.** 별도의 조작이 필요 없습니다.

#### 무엇이 달라지나?
- 기존: 정규식 기반 → 가끔 색상이 틀리거나 누락됨
- treesitter: AST(구문 트리) 기반 → 정확한 구문 강조
  - 함수 이름, 변수, 타입, 매크로 등이 각각 다른 색상으로 표시
  - 문자열 안의 이스케이프 시퀀스도 정확히 강조
  - 주석, 키워드, 리터럴 등 구분이 더 명확

#### 관리 명령어

```vim
:TSInstall <언어>        " 언어 파서 설치 (예: :TSInstall rust)
:TSUpdate               " 모든 파서 업데이트
:TSInstallInfo          " 설치된 파서 목록
```

### 3-9. indent-blankline.nvim (들여쓰기 가이드라인)

> **한 줄 설명:** 들여쓰기 레벨마다 세로 줄을 그려서 코드 구조를 시각화

#### 어떤 상황에서 쓰나?
- **자동으로 동작합니다.** 별도의 조작이 필요 없습니다.

### 3-10. lualine.nvim (상태바)

> **한 줄 설명:** 화면 하단에 유용한 정보를 표시하는 상태바

#### 어떤 상황에서 쓰나?
- **자동으로 동작합니다.** 항상 화면 하단에 표시됩니다.

#### 상태바에 표시되는 정보

```
┌──────────────────────────────────────────────────────────────────┐
│ NORMAL │ main │ +3 ~1 -0 │ file.cpp     │ 2E 1W │ utf-8 │ cpp │ 45% │ 123:15 │
└──────────────────────────────────────────────────────────────────┘
  모드    브랜치   diff 통계   파일명       진단     인코딩  타입  진행률  줄:열
```

| 영역 | 표시 내용 |
|------|----------|
| 모드 | 현재 모드 (NORMAL, INSERT, VISUAL 등) - 색상으로도 구분 |
| 브랜치 | 현재 git 브랜치 이름 |
| diff 통계 | 추가(+)/수정(~)/삭제(-) 줄 수 |
| 파일명 | 현재 편집 중인 파일 이름 |
| 진단 | LSP 에러(E)/경고(W) 수 |
| 인코딩 | 파일 인코딩 (utf-8 등) |
| 타입 | 파일 타입 (cpp, python 등) |
| 진행률 | 파일에서의 위치 (퍼센트) |
| 줄:열 | 현재 커서의 줄 번호와 열 번호 |

#### 상단 탭바

상태바 외에 **상단에도 탭바**가 표시됩니다:
```
┌──────────────────────────────────────────────────────────────────┐
│ [1] file1.cpp   [2] file2.h   [3] CMakeLists.txt               │
└──────────────────────────────────────────────────────────────────┘
```

열린 버퍼(파일) 목록이 표시되어 현재 어떤 파일들이 열려있는지 한눈에 볼 수 있습니다.
`,q`와 `,w`로 이전/다음 버퍼로 전환합니다.

## 4. 실전 워크플로우 예시

### 4-1. HANA 소스에서 특정 함수 찾아서 정의 보기

**목표:** `getColumnValue` 함수가 어떻게 구현되어 있는지 확인

```
1. nvim 실행 (아무 C++ 파일 열기)

2. ,fg 를 누름 (전체 텍스트 검색)

3. 검색창에 "getColumnValue" 입력
   → 이 문자열이 포함된 모든 파일과 줄이 목록으로 표시됨
   → 오른쪽에 미리보기로 해당 줄 주변 코드가 보임

4. 원하는 결과를 Ctrl+j/k로 이동, Enter로 열기

5. 함수 호출 위에 커서를 놓고 gd
   → 함수 정의로 점프!

6. 정의를 읽고 나서 Ctrl+o 로 이전 위치로 돌아오기

7. gr 로 이 함수를 호출하는 모든 곳 확인

8. K 로 함수의 타입 시그니처 및 문서 확인
```

### 4-2. git 변경사항 확인하고 커밋하기

**목표:** 코드 수정 후 변경사항을 확인하고 커밋 만들기

```
1. 코드를 수정합니다
   → 왼쪽에 초록/파란 막대가 표시됨 (gitsigns)

2. ]c 로 변경된 곳들을 순서대로 방문

3. ,hp 로 각 변경의 before/after를 확인

4. 잘못 수정한 곳이 있다면 ,hr 로 해당 hunk만 복원

5. :Git 을 입력하여 fugitive의 status 화면 열기
   → 수정된 파일 목록이 표시됨

6. 스테이지할 파일 위에서 s 를 눌러 스테이지

7. cc 를 눌러 커밋 메시지 작성
   → 커밋 메시지 편집 화면이 열림
   → 메시지를 작성하고 :wq 로 저장하면 커밋 완료

8. q 로 status 화면 닫기
```

### 4-3. 여러 파일 동시에 열어서 작업하기

**목표:** 헤더 파일과 구현 파일을 동시에 보면서 작업

```
1. nvim src/MyClass.cpp    (구현 파일 열기)

2. :vsp src/MyClass.h       (헤더 파일을 오른쪽에 수직 분할로 열기)
   → 화면이 좌우로 나뉘어 두 파일이 동시에 보임

3. Ctrl+h / Ctrl+l 로 좌우 창 사이를 이동

4. 추가로 파일을 열고 싶으면:
   - ,ff 로 파일 검색
   - Ctrl+v 로 수직 분할로 열기 (또는 Ctrl+x로 수평)

5. 창 하나를 닫으려면:
   - 닫으려는 창으로 이동 후 Ctrl+w q
   - 또는 :q

6. 버퍼(탭) 간 전환:
   - ,q (이전 버퍼) / ,w (다음 버퍼)
   - 상단 탭바에서 현재 위치 확인

7. 모든 창을 닫고 하나만 보려면:
   - :only 또는 Ctrl+w o
```

### 유용한 꿀팁

1. **`.` (점)**: 마지막 편집 명령을 반복. `dd`로 줄 삭제 후 `.`을 누르면 또 삭제
2. **`Ctrl+o`**: 이전 위치로 돌아가기 (gd로 점프한 후 등)
3. **`Ctrl+i`**: 다음 위치로 이동 (Ctrl+o의 반대)
4. **`ciw`**: 커서 아래 단어를 삭제하고 Insert 모드 진입 (Change Inner Word)
5. **`di"` / `di(`**: 따옴표/괄호 안의 내용 삭제
6. **`va{`**: 중괄호 블록 전체 선택 (Visual Around {)
7. **숫자 + 동작**: `5j` = 5줄 아래로, `3dd` = 3줄 삭제, `10w` = 10단어 앞으로
